# Semantic Retrieval System: Methodology & Experimental Setup

## 1. Introduction to the Retrieval System

This project implements a **Semantic Search** system tailored for a **Question Answering (QA)** task. The primary objective is to retrieve text fragments (*candidate chunks*) that contain the correct answer to a specific *query* expressed in natural language, from a given set of candidates.

The pipeline leverages **Transformer-based models** to encode both queries and texts into dense embeddings within a shared vector space, enabling semantic similarity matching.

---

## 2. Methodology and Implemented Functions

To manage the **encoding**, **ranking**, and **evaluation** processes efficiently, the codebase has been modularized into the following functions.

### Function Overview

| Function Name           | Role                                            | Key Mechanism                                                                  |
| :---------------------- | :---------------------------------------------- | :----------------------------------------------------------------------------- |
| **`mean_pooling`**      | Extract sentence embeddings from token outputs. | Applies attention mask to zero out padding, then averages the vector outputs.  |
| **`encode_texts`**      | Batch-transforms text strings into vectors.     | Handles tokenization, disables gradient calculation, and applies pooling.      |
| **`cosine_similarity`** | Computes vector proximity (Query vs. Chunks).   | Normalizes vectors and computes dot products via `torch.matmul`.               |
| **`rank_chunks`**       | Generates the ordered ranking of candidates.    | Aggregates encoding and scoring, sorts results in descending order.            |
| **`hit_at_k`**          | Evaluates retrieval performance (Hit@k).        | Checks if the correct index exists within the top-$k$ retrieved results.       |
| **`evaluate`**          | Automates testing across dataset splits.        | Iterates over the dataset (e.g., Test Set) and aggregates Hit metrics.         |

#### Detailed Function Specifications

> **`mean_pooling(model_output, attention_mask)`**
> - **Purpose:** Extract a **sentence embedding** (a single vector representing the entire phrase's meaning) from individual tokens.
> - **Mechanism:** Models like `distilbert-base-uncased` return an embedding for every token in the sentence. This function multiplies the embeddings by the attention mask (which distinguishes real words from padding) and calculates the mean of the remaining vectors. This yields a compact, fixed-length vector representation for the entire sequence.

> **`encode_texts(texts, batch_size)`**
> - **Purpose:** Efficiently transform text strings into mathematical vectors.
> - **Mechanism:** The function splits the list of texts into blocks (batches) to optimize GPU memory (VRAM) usage. It uses the tokenizer to process the input, passes the data to the model with gradients disabled (`torch.no_grad()`), and finally applies the `mean_pooling`. The resulting vectors are concatenated and returned as a single tensor.

> **`cosine_similarity(query_emb, chunk_embs)`**
> - **Purpose:** Measure the semantic proximity between the query and the various text fragments.
> - **Mechanism:** The metric used is **cosine similarity**. For maximum efficiency in PyTorch, instead of using `for` loops, this function first normalizes all vectors (dividing by their norm to have a unit modulus) and then computes the dot product via matrix multiplication (`torch.matmul`). The result is a score array between -1 and 1.

> **`rank_chunks(query, chunks)`**
> - **Purpose:** Generate the ranking of text chunks.
> - **Mechanism:** Acts as an aggregator for a single example. It encodes the query and all candidate chunks, calculates scores using `cosine_similarity`, and uses `torch.argsort` to sort the scores in descending order. It returns a list of indices of the chunks ordered from most to least relevant.

> **`hit_at_k(ranking, correct_idx, k)`**
> - **Purpose:** Implement the mathematical logic of the **Hit@k** metric required for evaluation.
> - **Mechanism:** Leveraging Python's native slicing (`ranking[:k]`), this function isolates the first $k$ positions of the ranking generated by the model. It then verifies whether the index corresponding to the correct answer (`correct_idx`) exists within this subset. It returns `1` if the index is present (success) or `0` if absent.

> **`evaluate(dataset)`**
> - **Purpose:** Automate the testing of the model on entire dataset splits.
> - **Mechanism:** Iterates over all provided examples (e.g., the Test set). For each element, it extracts the query, candidates, and correct answer, calculates the ranking, and updates three separate counters for **Hit@1**, **Hit@3**, and **Hit@5**. Finally, it returns the accumulated totals to allow the calculation of final average percentages.
